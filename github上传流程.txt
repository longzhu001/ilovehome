1.在github上创建仓库,仓库名A
2.来到本地文件随便进入一个目录,git clone 仓库地址,这个时候目录下面就会多了一个仓库名A 的文件夹
3.进入到仓库文件夹里面
'''
git status 查看当前工作区域的文件,是否有为上传的文件,是本地的,并且知道位于哪个分区,议案是master,所以在这里的操作都是本地的,和远程独立分开的,没有关系
git branch 分支名   创建分支
git branch -d  分支名   删除分支
git checkout 分支名   切换分支(默认是master)
git add 提交文件到暂缓区 ,后面可以加目录名,文件名,当前文件全部文件
git reset HEAD 文件名, 把放到了暂缓区里的文件,重新拿回来,不提交了
分为三个区域    工作区   暂缓区   本地代码库
'''
4.git branch 分支名B
5.切换到自己的分支,进行操作
6.开始把文件放在仓库A文件夹里面,这个时候再查看状态就可以看到未上传的文件,为红色
7.通过 git add 文件名 ,把对应想上传的文件,从工作区提交到暂缓区域里,还没有到代码库里,其中可以是git add . 代表全部文件上传
8.git commit -m "说明这个文件的代码作用的提示,不能不写" ,这个时候,就会把暂缓区里面的文件上传到本地代码库,还不是远程代码库
9.git push origin 我的工作分支名B:远程的工作分支   一般都是一样的
10.执行上面的操作的时候,回弹出要输入用户名,就是邮箱,还有输入密码
11.去到远程里,就会出现你新建的文件,,然后点击右边的compar pull request
12.合并 开始去merge pull request 是把自己上传的代码,进行合并到最终的目录仓库里,就会出现提交的文件了
13.git pull origin master   保证你开发的目录里的文件,永远是最新的
